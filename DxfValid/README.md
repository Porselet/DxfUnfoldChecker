# DxfValid — Валидатор DXF-разверток листовых деталей из Siemens NX

Промышленный инструмент автоматического контроля геометрии и топологии плоских разверток (Flat Pattern) листового металла перед их отправкой на лазерные станки с ЧПУ. Программа выявляет технологический брак чертежей, защищая лазерную голову от повреждений, а производство — от прямых убытков.

## 🚀 Основные возможности движка

*   **Гибридная всеядность версий DXF**: Основной скоростной движок на базе `netDxf` обрабатывает современные файлы (AutoCAD 2010 и новее), а резервный автоматический модуль на базе `IxMilia.Dxf` перехватывает и нативно разбирает устаревший формат `AutoCAD R12 (R13/R14)`, исключая падения программы по ошибке структуры.
*   **Линеаризация сплайнов из Siemens NX**: Движок автоматически подхватывает скрытые B-сплайны (кривые высшего порядка, которых в развертках NX может быть более 700 штук), аппроксимирует их методом `ToPolyline2D(10)` в цепочки точных хорд и включает в общий контур реза.
*   **Интеллектуальное адаптивное дробление дуг**: Защищает мелкие технологические подрезы гибов (Corner Reliefs) от ложного удаления. Микро-дуги делятся всего на 2–4 сегмента, сохраняя длину хорды выше допуска сборщика графа.
*   **Защита от фантомных наложений**: Полуокружности (круглые пазы/отверстия) анализируются по методу излома средней точки дуги, что исключает ложные тригонометрические коллизии на встречных векторах.
*   **Многопоточность и асинхронный UI**: Весь пакетный анализ тяжелой геометрии с медленных заводских флешек или сетевых накопителей вынесен в фоновый поток `Task.Run`. Интерфейс WPF защищен от замерзания (статуса *«Не отвечает»*) и оснащен модальным окном прогресса `ProgressBar` с выводом имени текущего файла в реальном времени.

## 📐 Архитектура проекта

Проект спроектирован по модульному принципу «Пути В» и жестко разделен на две независимые части:

1.  **`DxfValid.Core.dll` (Класс-Библиотека / SDK-компонент)**: Автономный вычислительный мозг. Содержит только чистую векторную математику, парсеры, сборщик графов и инспектор наложений. Не привязан к WPF и Windows Forms. Может быть встроен в макросы Компаса, Siemens NX, AutoCAD или консольные скрипты автоматизации фабрики.
2.  **`DxfValid.exe` (WPF-Интерфейс)**: Графическая оболочка для инженера ОТК/технолога. Отвечает за диспетчеризацию папок, интерактивный рендеринг геометрии на холсте `Canvas`, масштабирование мыши (Auto-Fit) и управление контекстными командами Windows.

## 🛠️ Инструкция по работе с интерфейсом

### Пакетная валидация папок
1. Запустите приложение и нажмите кнопку **«Выбрать папку»** в верхнем углу.
2. Укажите корневую директорию комплекта. Сканирование происходит **рекурсивно по всем вложенным подпапкам**.
3. На время чтения на экране появится защищенное модальное окно прогресса. Клик по крестику блокируется до окончания работы фонового потока.
4. По окончании вся аналитика выводится в таблицу со следующими статусами:
   * 🟢 **ГОДЕН**: Чертеж математически идеален и замкнут.
   * ⚠️ **ЗАМЕЧАНИЯ/ПРЕДУПРЕЖДЕНИЯ**: Контур замкнут, но обнаружен микро-мусор, произведено выпрямление вырожденных дуг из NX (R > 10 метров), либо файл сохранен в устаревшем формате AutoCAD R12 (требуется пересохранение).
   * ❌ **БРАК**: Обнаружены физические разомкнутые контуры (зазоры) или наложения линий (лазер пройдет по одному месту дважды).

### Графический интерактивный просмотр
Дважды кликните по любой строке в таблице, чтобы открыть окно детального просмотра геометрии:
*   Программа автоматически рассчитает габариты детали и выведет её **строго по центру темного экрана (Auto-Fit)** в оптимальном масштабе.
*   Векторы контура отрисовываются серым цветом, места наложений подсвечиваются бирюзовым, а точки разомкнутых зазоров — жирными красными маркерами.
*   Управление камерой: **Колесико мыши** — плавный зум в точку курсора без скачков координат, **Зажатая левая кнопка мыши** — панорамирование (перетаскивание детали).
*   **Горячая клавиша [Пробел] или [F]**: Мгновенный сброс камеры и возврат чертежа ровно по центру экрана, если вы укрутили деталь далеко в сторону.

### Smart Sync (Пакетная локализация брака)
Если в таблице найдено много бракованных деталей:
1. Отсортируйте список по колонке «Статус».
2. Зажмите **Ctrl** или **Shift** и выделите мышкой сразу несколько проблемных строк.
3. Кликните правой кнопкой мыши по таблице и выберите **«📁 Открыть папки с выделением файлов (Smart Sync)»**.
4. Программа сгруппирует файлы по папкам через LINQ и откроет встроенный Проводник Windows (`explorer.exe`) по умному алгоритму: на каждую вовлеченную подпапку откроется **всего одно окно**, внутри которого **пачкой выделятся сразу все нужные файлы** (через Win32 API `SHOpenFolderAndSelectItems`). Их можно в один клик перетащить в архив для отправки конструкторам.

## 💻 Ключи командной строки (CLI Автоматизация)

Программу `DxfValid.exe` можно вызывать из сторонних скриптов, плагинов Siemens NX или макросов САПР в автоматическом графическом режиме.

### 1. Автоматическая проверка всей папки
При передаче ключа папки программа мгновенно распахивает главное окно, сама запускает фоновое асинхронное сканирование и выводит на экран таблицу результатов без единого клика со стороны человека.
```bash
DxfValid.exe -folder "C:\Storage\NX_Unfolds_Batch"
```

### 2. Точечная проверка одного чертежа
Если вам нужно проверить только одну деталь, главное окно полностью игнорируется и не создается в памяти, чтобы не засорять рабочий стол. Программа мгновенно открывает **только окно графического просмотра дефектов** для этого конкретного DXF-файла. Как только инженер закрывает окно крестиком, процесс чисто выгружается из оперативной памяти.
```bash
DxfValid.exe -file "C:\Storage\NX_Unfolds_Batch\Detail_Profile.dxf"
```

## 🛠️ Разработчику: Быстрый старт со встроенным SDK

Для интеграции аналитического движка в ваши собственные C# консольные утилиты или веб-сервисы, подключите скомпилированную библиотеку `DxfValid.Core.dll` к проекту и используйте класс пофайловой валидации `DxfSingleFileValidator`:

```csharp
using DxfValid.Core;

// Инициализируем изолированный валидатор одного файла
var validator = new DxfSingleFileValidator();

// Запускаем анализ чертежа с инженерным допуском стыковки графа 0.05 мм
FileValidationSummary report = validator.ValidateFile(@"C:\Detail.dxf", tolerance: 0.05);

// Проверяем вердикт программно
if (report.Status == FileValidationStatus.Defective)
{
    Console.WriteLine(\$"Найдено зазоров: {report.Topology.OpenContours.Count}");
    Console.WriteLine(\$"Найдено наложений: {report.Cleanliness.OverlappingDefects.Count}");
}
```
